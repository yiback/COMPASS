> **관련 문서**: [개발요구사항](./개발요구사항.md), [기술스택](./기술스택.md)

## 1. 프로젝트 핵심

**목적**: 학원이 AI로 학교별 맞춤 예상시험을 생성하고 학생 취약점을 분석하는 플랫폼

**타겟**: 소형~중형 보습학원 (중등 수학 담당)

---

## 2. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | COMPASS (교육 나침반) |
| 비즈니스 모델 | B2B2C (학원 → 학생) |
| MVP 범위 | 중등 수학 전체 (중1, 중2, 중3) |
| Phase 0 검증 | 중3 수학 |
| 핵심 가치 | AI 기반 학교별 예상시험 생성 + 취약점 분석 |

---

## 3. 개발 Phase 정의

### Phase 0: AI 문제 출제 검증

| 단계 | 기능 | 검증 방법 | 목표 |
|------|------|----------|------|
| 0-1 | AI 문제 출제 (학년/단원 기반) + 프로토타입 UI | 교사 검수 | AI 출제 가능성 검증 |
| 0-2 | 교육부 학습목표(성취기준) 관리 기능 | 기능 테스트 | 성취기준 DB 구축 |
| 0-3 | 성취기준 기반 AI 출제 + 검증 | 교사 검수 | 교육과정 연계 출제 |

**성공 기준**:
- 정답 정확도: 98% 이상
- 성취기준 매핑 정확도: 95% 이상
- 검수 후 수정 필요 비율: 30% 이하

### Phase 1: 데이터 기반 구축

| 단계 | 기능 | 검증 방법 | 목표 |
|------|------|----------|------|
| 1-1 | 기출문제 관리 기능 | 기능 테스트 | 학교별 기출 DB 구축 |
| 1-2 | 학교/학원/학생 관리 기능 | 기능 테스트 | 사용자 데이터 구조화 |
| 1-3 | 기출문제 참고 AI 출제 + 검증 | 교사 검수 | 학교별 맞춤 출제 |

### Phase 2: 핵심 서비스

| 단계 | 기능 | 검증 방법 | 목표 |
|------|------|----------|------|
| 2-1 | 학교별 예상 문제 출제 (학습목표 + 기출 분석) | 교사 검수 | 핵심 가치 제공 |
| 2-2 | 시험 응시 (종이 시험지 기반) | 사용자 테스트 | 학원 현장 적용 |
| 2-3 | AI 채점 + 교사 검수 | 정확도 측정 | 채점 자동화 |
| 2-4 | 오답 분석 + 대시보드 | 사용자 피드백 | 학습 인사이트 제공 |

### Phase 3+: 확장 (MVP 이후)

| 기능 | 설명 | 우선순위 |
|------|------|---------|
| 온라인 시험 응시 | 웹/모바일 직접 응시 | 높음 |
| 결제 시스템 | 구독 자동화 | 높음 |
| 학부모 기능 | 계정, 알림, 대시보드 | 중간 |
| 교사 출제 성향 분석 | 학교별 교사 패턴 | 낮음 |

## 4. 사용자 여정

### Phase 0: AI 검증 흐름

```
학년/단원 선택 → AI 문제 생성 → 교사 검수 → 품질 확인 → 승인/반려
```

### Phase 1: 데이터 구축 흐름

```
학습목표 등록 → 기출문제 등록 (이미지/OCR) → 학교/학원/학생 등록 → 기출 기반 AI 출제
```

### Phase 2: 핵심 서비스 흐름

```
학교별 예상시험 생성 → 시험지 PDF 출력 → 학생 응시 (종이)
→ 답안지 촬영 → AI 채점 + 교사 검수 → 오답 분석 → 리포트
```

## 5. 사용자 역할 및 권한

### 5.1 역할 정의

| 역할 | 설명 | Phase |
|------|------|-------|
| 학생 | 시험 응시, 성적 확인, 오답 학습 | Phase 2 |
| 교사 | 문제 관리, 시험 출제, 채점 검수, 학생 관리 | Phase 0 |
| 학원 관리자 | 학원 설정, 교사/학생 관리, 통계 확인 | Phase 1 |
| 시스템 관리자 | 학습목표 관리, 전체 시스템 관리 | Phase 0 |
| 학부모 | 자녀 성적 확인 (간접 접근) | Phase 3 |

### 5.2 역할별 주요 기능

**학생**:
- 내 시험 목록 확인
- 답안 제출 (사진 촬영)
- 채점 결과 확인
- 오답 노트 확인
- 취약점 분석 확인

**교사**:
- AI 문제 생성 요청
- AI 생성 문제 검수 (승인/반려/수정)
- 기출문제 등록 및 관리
- 시험 생성 및 배포
- AI 채점 결과 검수 및 오버라이드
- 담당 학생 관리

**학원 관리자**:
- 학원 정보 설정
- 교사 관리 (초대, 권한)
- 학생 관리 (등록, 소속 학교)
- 학원 전체 통계 대시보드
- 학부모 리포트 PDF 내보내기

## 8. 페이지별 상세 기능

### 8.1 Phase 0 페이지

#### 문제 생성 페이지 (프로토타입)

**목적**: AI를 통한 문제 생성 요청

**UI 요소**:
- 학년 선택 드롭다운 (중1, 중2, 중3)
- 과목 선택 (수학 - MVP에서 고정)
- 단원 선택 (트리 구조)
- 성취기준 선택 (선택적)
- 문제 유형 선택 (객관식, 단답형, 서술형)
- 난이도 선택 (상/중/하)
- 문제 개수 입력
- 생성 버튼

**동작**:
1. 조건 입력 → AI 생성 요청
2. 로딩 상태 표시 (예상 시간 안내)
3. 생성 완료 → 문제 미리보기
4. 검수 요청 버튼

#### 문제 검수 페이지

**목적**: AI 생성 문제의 품질 검수

**UI 요소**:
- 검수 대기 목록 (카드/테이블 뷰)
- 문제 상세 보기 (문제, 보기, 정답, 해설)
- 검수 액션 버튼:
  - 승인: 문제 DB에 저장
  - 반려: 반려 사유 입력
  - 수정: 인라인 에디터로 수정 후 승인
- 수학 수식 렌더링 (LaTeX)
- 이미지 첨부 지원

**검수 기준 체크리스트**:
- [ ] 문제가 수학적으로 정확한가?
- [ ] 정답이 올바른가?
- [ ] 해설이 이해하기 쉬운가?
- [ ] 난이도가 적절한가?
- [ ] 성취기준과 일치하는가?

#### 학습목표 관리 페이지

**목적**: 교육부 성취기준 관리

**UI 요소**:
- 성취기준 트리 뷰 (학년 > 단원 > 성취기준)
- 검색/필터 기능
- 성취기준 상세:
  - 코드, 내용, 학년, 단원, 키워드
- CRUD 폼

### 8.2 Phase 1 페이지

#### 기출문제 관리 페이지

**목적**: 학교별 기출문제 등록 및 관리

**UI 요소**:
- 이미지 업로드 영역 (드래그앤드롭)
- OCR 결과 미리보기
- 메타데이터 입력:
  - 학교 선택 (검색)
  - 학년/학기
  - 시험 유형 (중간/기말)
  - 출제 연도
- 문제별 성취기준 매핑 (AI 추천 + 수동 선택)
- 저작권 출처 표시

**OCR 처리 흐름**:
1. 이미지 업로드
2. Mathpix (수학 수식) + Google Vision (한글) 처리
3. 추출 결과 미리보기
4. 수동 보정
5. 저장

#### 학교/학원 관리 페이지

**목적**: 학교 및 학원 정보 관리

**학교 관리**:
- 학교 검색 (교육부 데이터 연동)
- 학교 정보 조회 (주소, 학교 유형)
- 관심 학교 등록

**학원 설정**:
- 학원 기본 정보 (이름, 주소, 연락처)
- 브랜딩 설정 (로고, 색상)
- 운영 정보 (수업 시간, 정원)

- 학생 삭제/비활성화

**교사 관리**:
- 교사 목록
- 교사 초대 (이메일)
- 담당 학년/과목 설정
- 권한 관리

**회원가입**:

- 학원 신규 등록

## 9. 데이터 모델

### 9.1 엔티티 목록

| 엔티티 | 설명 | 주요 필드 |
|--------|------|----------|
| User | 사용자 기본 정보 | id, email, name, role, academy_id |
| Academy | 학원/기관 | id, name, address, logo, settings |
| School | 학교 | id, name, type, region, address |
| Student | 학생 확장 정보 | id, user_id, school_id, grade |
| Teacher | 교사 확장 정보 | id, user_id, subjects, grades |
| AchievementStandard | 성취기준 | id, code, content, grade, unit, subject |
| Question | 문제 | id, content, type, answer, explanation, is_ai_generated, ai_review_status |
| PastExamQuestion | 기출문제 | id, school_id, year, semester, exam_type, source_image |
| Exam | 시험 | id, title, school_id, grade, questions, duration |
| ExamResult | 시험 결과 | id, exam_id, student_id, score, submitted_at |
| Answer | 답안 | id, exam_result_id, question_id, student_answer, is_correct, ai_confidence |
| WrongAnswer | 오답 기록 | id, student_id, question_id, reviewed |

### 9.2 주요 관계

```
Academy (1) ─────── (*) User
User (1) ─────────── (0..1) Student
User (1) ─────────── (0..1) Teacher
Student (*) ──────── (1) School
School (1) ────────── (*) PastExamQuestion
AchievementStandard (1) ─── (*) Question
Question (*) ──────── (*) Exam (via ExamQuestion)
Exam (1) ──────────── (*) ExamResult
ExamResult (1) ────── (*) Answer
Student (1) ────────── (*) WrongAnswer
```

### 9.3 AI 생성 문제 관련 필드

```typescript
interface Question {
  id: string;
  content: string;           // 문제 내용
  type: QuestionType;        // 객관식 | 단답형 | 서술형
  options?: string[];        // 객관식 보기
  answer: string;            // 정답
  explanation: string;       // 해설
  difficulty: number;        // 난이도 (1-5)

  // AI 생성 관련
  is_ai_generated: boolean;           // AI 생성 여부
  ai_review_status: ReviewStatus;     // pending | approved | rejected
  ai_model: string;                   // 생성 AI 모델 (gemini, gpt-4o 등)
  ai_generation: number;              // AI 세대 (자기참조 추적)
  reviewed_by?: string;               // 검수자 ID
  reviewed_at?: Date;                 // 검수 일시

  // 교육과정 연계
  achievement_standard_id: string;    // 성취기준 연결
  grade: number;                      // 학년
  unit: string;                       // 단원

  // 메타데이터
  source?: QuestionSource;            // 출처 정보
  created_at: Date;
  updated_at: Date;
}

enum ReviewStatus {
  PENDING = 'pending',
  APPROVED = 'approved',
  REJECTED = 'rejected'
}

interface QuestionSource {
  type: 'past_exam' | 'textbook' | 'self_made' | 'ai_generated';
  school?: string;
  year?: number;
  copyright_status: 'public' | 'licensed' | 'owned';
}
```

## 11. 운영 정책 (AI 윤리)

### 11.1 교사 검수 필수 (HITL - Human in the Loop)

| 항목 | 정책 |
|------|------|
| AI 문제 생성 | 모든 AI 생성 문제는 교사 검수 후 사용 |
| AI 채점 | 채점 신뢰도 80% 미만은 교사 검수 필수 |
| 검수 기준 | 수학적 정확성, 교육적 적절성, 난이도 확인 |

### 11.2 이의제기 시스템

| 단계 | 내용 |
|------|------|
| 1단계 | 학생이 채점 결과에 이의 제기 |
| 2단계 | 교사가 해당 답안 재검토 |
| 3단계 | 오채점 확인 시 점수 수정 |
| 4단계 | 이의 제기 이력 기록 |

### 11.3 AI 오판 시 학생 불이익 금지 원칙

- AI 오류로 인한 성적 불이익 금지
- 의심 시 학생에게 유리하게 처리
- 교사의 최종 판단권 보장

### 11.4 데이터 보관 정책

| 데이터 유형 | 보관 기간 | 비고 |
|------------|----------|------|
| 학습 이력 | 5년 | 성적 추이 분석용 |
| 성적 데이터 | 7년 | 법적 요건 고려 |
| 원본 답안 이미지 | 1년 | 이의 제기 대응 |
| 퇴원 학생 | 퇴원 후 1년 | 익명화 또는 삭제 |

### 11.5 AI 자기참조 정책 (모델 붕괴 방지)

| 조치 | 내용 |
|------|------|
| 원본 데이터 비율 | 70% 이상 유지 (인간 생성 문제) |
| HITL 검수 필수 | 교사 승인 없이 참조 데이터 추가 금지 |
| 다양성 모니터링 | 분기별 난이도/유형/단원 분포 점검 |
| 세대 추적 | AI 생성 세대(generation) 기록 |
