## 6. 핵심 기술 교훈 Top 5

1. **thenable mock 패턴**: Supabase 쿼리 빌더는 Promise가 아닌 thenable. `Object.defineProperty(mockQuery, 'then', ...)` 으로 어떤 체인 순서든 `await` 시 결과 반환. 동적 필터 조합 테스트 필수
2. **트리거 vs 제어 모드 Dialog**: 생성(DialogTrigger 기반)과 수정(부모 open/onOpenChange 제어)을 union type props로 하나의 컴포넌트에서 처리. DataTable row action에서 제어 모드 필수
3. **UPSERT 불변 필드 분리**: `ON CONFLICT DO UPDATE`에서 subject/grade/semester를 갱신 대상에서 제외 = Zod updateSchema + UI disabled 필드와 동일 원칙 ("변하는 것과 변하지 않는 것의 분리")
4. **JSON.parse try/catch 필수**: FormData에서 전달된 JSON 문자열은 항상 파싱 실패 가능. 에러 시 500 대신 친절한 메시지 반환. create + update 양쪽 모두 적용
5. **Defense in Depth 3계층**: page.tsx(requireRole) + Server Action(checkSystemAdminRole) + RLS(has_role) — 2번 생략하면 Direct POST 공격에 DB까지 도달 → 불친절한 에러 + 불필요한 부하
