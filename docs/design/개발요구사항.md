## 기존 요구사항 (1-5)

### 1. 학년별 과목별 교육부 학습목표 관리
- 각 학년별, 각 과목별로 교육부 학습목표를 항상 최신으로 관리한다.

### 2. 각 학교별 기출 문제 관리
- **중등:** 각 학교별로 각 과목들의 기출문제를 관리한다.
- **초등:** 교육청 학력평가, 단원평가 문제를 관리한다. (초등은 2013년부터 정기고사 폐지)
- 기출문제들이 교육부의 학습목표(성취기준)와 어떻게 연관되는지 분석한다. (AI 활용)
- 기출문제를 출제한 교사들의 출제 경향을 분석한다. (AI 활용) - **중등 한정**

### 3. 학년별 과목별 평가 출제 엔진
- **중등:** 1과 2를 바탕으로 특정학교의 예상시험을 출제한다. (AI 활용)
- **초등:** 성취기준 기반 진단평가/단원평가를 출제한다. (AI 활용)

### 4. 학생 관리
- 학생의 소속, 학년 등을 관리한다.

### 5. 학생 학업 성취도 관리
- 학생의 과목별 학습 성취도를 지역, 전국 단위로 관리한다. (AI 활용)
- 학생의 과목별 성취도에 따른 장단점을 분석한다. (AI 활용)
- 학생의 과목별 학습 방향을 제시하고 관리한다. (AI 활용)

### 7. 시험 응시 및 채점 시스템

#### 시험 응시 방식 (종이 시험지 기반)

**기본 흐름:**
```
시험지 PDF 출력 → 학생이 문제지에 직접 풀이 → 문제지 사진 촬영 → AI 이미지 인식 → 채점
```

**시험 응시 환경:**
- 시험지 PDF 출력 기능 (학교별 출제 스타일 반영)
- 학생이 문제지에 직접 정답 표시/풀이 작성 (실제 학교 시험과 동일)
- 문제지 전체를 사진 촬영하여 제출
- 시험 시간 관리 (타이머)

#### 기출문제 이미지 입력

- 이미지 형태의 기출문제 OCR/이미지 인식으로 등록
- 문제, 보기, 정답 영역 자동 분리
- 수학 기호/수식 인식 (Math-OCR)

#### AI 이미지 인식 채점

**채점 범위별 접근:**
| 문제 유형 | MVP (Phase 1) | Phase 2 | Phase 3 |
|----------|---------------|---------|---------|
| 객관식 (마킹) | AI 인식 + 교사 검수 | AI 자동화 확대 | AI 자동 |
| 단답형 | AI 인식 + 교사 검수 필수 | AI + 표본 검수 | AI 자동 |
| 서술형/풀이 | 교사 채점 (AI 보조) | AI 채점 "참고용" | AI 확대 (검증 후) |

**AI 인식 기술 요소:**
- 레이아웃 인식: 문제지에서 각 문제 영역 파악
- 마킹 인식: 동그라미, 체크, 밑줄 등 표시 인식
- 손글씨 인식 (HWR): 숫자, 단어, 수식 인식
- 이미지 전처리: 그림자 제거, 왜곡 보정, 명암비 조절

#### AI 이미지 채점의 기술적 한계 및 리스크

**식별된 문제점:**
| 문제 | 기술적 해결 | 심각도 |
|------|------------|--------|
| 이미지 품질 (조명, 흔들림) | ⭕ 전처리로 상당 부분 해결 | 중간 |
| 손글씨 다양성 (악필, 흘림체) | △ 부분적 해결 | 높음 |
| 수학 기호 (분수, 루트, 지수) | △ 부분적 해결 | 높음 |
| 유사 기호 혼동 (0/O, 1/l, 2/Z) | △ 부분적 해결 | 높음 |
| 수정 흔적 (지우개, 덧씌우기) | △ 부분적 해결 | 중간 |
| False Negative (정답→오답) | ❌ 완전 해결 불가 | **매우 높음** |
| 형평성 (디바이스/필체 격차) | ❌ 기술로 해결 불가 | **매우 높음** |

**가장 심각한 리스크: False Negative**
- 정답인데 AI가 오답으로 처리 → 학생 좌절감, 시스템 불신
- 학습 능력 외 요인(필체, 디바이스)으로 불이익 → 교육적 불평등

#### 필수 보완책

| 구분 | 보완책 | 설명 |
|------|--------|------|
| 기술적 | 촬영 가이드 | 앱 내 실시간 이미지 품질 진단, 재촬영 유도 |
| 기술적 | 이미지 전처리 | 그림자 제거, 왜곡 보정, 명암비 자동 조절 |
| 운영적 | 교사 검수 (HITL) | AI 채점 결과를 교사가 검토/수정 (Human-in-the-Loop) |
| 운영적 | 이의제기 시스템 | 학생이 오채점에 이의 제기 → 교사 재검토 |
| 윤리적 | AI 오판 불이익 금지 | AI 오류 시 학생에게 불이익 없음 원칙 |
| 단계적 | 참고용 우선 | 초기에는 형성 평가/자율 학습용으로 제한 |

**윤리적 원칙:**
- AI 오판 시 학생에게 어떠한 불이익도 주지 않음
- 인간 교사의 최종 판단권 보장
- 투명한 이의제기 및 재심 절차 제공

#### 채점 및 해설
- 문제별 상세 해설 제공
- 오답 해설 및 관련 개념 연결
- 풀이 과정 피드백 (교사 또는 AI 보조)

#### 시험 이력 관리
- 응시 기록 저장 (원본 이미지 포함)
- 시험별 성적 추이 분석
- AI 채점 정확도 모니터링

### 12. 교육 콘텐츠 관리 시스템 (CMS)

#### 문제 관리
- 문제 등록/수정/삭제

- 난이도 체계 관리
- 교육과정 연계 태깅

#### 해설 및 힌트 관리
- 문제별 해설 작성

#### 검수 워크플로우
- 문제 품질 검수 프로세스
- 전문가 리뷰 시스템
- 버전 관리

#### 콘텐츠 확보 전략 (수정)

**기본 방향:** 자체 제작 우선 (AI + 내부 교육 전문가)

**샘플 문제 출처 (혼합 접근):**
| 출처 | 활용 | 저작권 리스크 |
|------|------|--------------|
| 교육청 공개 문제 | 기본 샘플 베이스 | 낮음 (공개 자료) |
| 시중 교재 | 유형/난이도 참고 | 중간 (변형 필요) |
| 내부 전문가 제작 | 부족한 유형 보완 | 없음 |

**검수 체계:**
- 내부 교육 전문가 채용 (과목당 1명)
  - 초등 수학 전문가: 문제 검수 + 성취기준 매핑
  - 초등 영어 전문가: 문제 검수 + 난이도 조정
  - 중등 수학 전문가: 문제 검수 + 출제 경향 분석
  - 중등 영어 전문가: 문제 검수 + 문법/어휘 검증

**워크플로우:**
```
샘플 문제 제시 → AI 유사 문제 출제 → 검수자 품질 평가 → 피드백 반영 → 문제 DB 등록
```

**장기 확장 전략:**
1. **기존 문제은행 라이선스/인수** - 한국 교육 출판사와 파트너십 (규모 확대 시)
2. **교사 크라우드소싱** - 플랫폼 안정화 후 도입, 품질 관리 워크플로우 필수

### 14. 비기능적 요구사항

#### 플랫폼
- 웹 앱 / 모바일 앱 지원
- 반응형 디자인
- 오프라인 기능: **MVP는 정적 콘텐츠 캐싱만, Phase 2에서 제한적 오프라인 응시**

#### 성능 목표
| 지표 | MVP 목표 | 상용화 목표 | 장기 목표 |
|------|---------|-----------|----------|
| 동시 접속자 | 500명 | 2,000~5,000명 | 50,000명+ |
| API 응답 시간 | P90 500ms | P90 300ms | P90 200ms |
| AI 문제 생성 | 5초 이내 | 3초 이내 | 2초 이내 |
| 페이지 로딩 | 3초 이내 | 2초 이내 | 1초 이내 |

*주: 피크 타임(18시~22시) 집중 사용 고려 필요*

#### 데이터 관리
| 항목 | 정책 |
|------|------|
| 데이터 보관 기간 | 학습 이력 5년, 성적 데이터 7년 |
| 아카이빙 | 3년 이상 경과 데이터 콜드 스토리지 이전 |
| 삭제 정책 | 퇴원 후 1년 유예, 이후 익명화 또는 삭제 |

#### 멀티테넌시 전략
- **방식:** 논리적 분리 (Logical Multi-tenancy)
- **구현:** 모든 테이블에 `tenant_id` 포함, 애플리케이션 레벨 강제
- **보안:** Row-Level Security (RLS) 적용 권장

#### 보안
- 데이터 암호화 (전송: TLS 1.3, 저장: AES-256)
- 기관별 데이터 분리 (tenant_id 기반)
- 접근 로그 관리 (최소 2년 보관)
- 개인정보 마스킹 (로그, 개발 환경)

### 20.2 AI 문제 출제 - 자기참조 정책 (핵심)

> ⚠️ **모델 붕괴(Model Collapse) 방지가 핵심 목표**

#### AI 생성 문제 → 참조 데이터 활용 프로세스
```
┌─────────────────────────────────────────────────────┐
│  1. AI가 문제 생성                                  │
│  2. is_ai_generated: true 플래그 설정               │
│  3. 교사 검수 (HITL - Human in the Loop)            │
│     - 수학적 정확성 검증                            │
│     - 교육적 적절성 평가                            │
│     - 난이도/유형 확인                              │
│  4. 검수 결과                                       │
│     - 승인 → ai_review_status: "approved"           │
│     - 반려 → ai_review_status: "rejected"           │
│  5. 승인된 문제만 참조 데이터로 승격                │
│  6. 비율 유지: 원본(인간) 70% + AI승인 30%          │
└─────────────────────────────────────────────────────┘
```

#### 모델 붕괴 방지 조치
| 조치 | 설명 |
|------|------|
| 원본 데이터 비율 | 70% 이상 유지 (인간 생성 문제) |
| HITL 검수 필수 | 교사 승인 없이 참조 데이터 추가 금지 |
| 다양성 모니터링 | 분기별 난이도/유형/단원 분포 점검 |
| 세대 추적 | AI 생성 세대(generation) 기록 |
| 외부 데이터 유입 | 품질 저하 감지 시 신규 데이터 추가 |

#### 품질 관리 지표
| 지표 | 목표 | 측정 주기 |
|------|------|----------|
| 교사 승인율 | 70% 이상 | 월간 |
| 다양성 지수 | 분포 편차 20% 이내 | 분기별 |
| 오류 발견율 | 5% 이하 | 월간 |
