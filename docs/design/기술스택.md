## 개요

이 문서는 Compass 프로젝트의 기술스택을 정의합니다. 개발요구사항을 기반으로 edu-system-architect와 edu-critical-analyst 에이전트 간의 토론을 통해 최적의 기술스택을 도출하였습니다.

### Phase 로드맵

```
Phase 0-1 (MVP, 현재)     Phase 2+ (스케일업, 계획)
━━━━━━━━━━━━━━━━━━━━━    ━━━━━━━━━━━━━━━━━━━━━━━
Next.js 15 풀스택          → NestJS 백엔드 분리
Supabase (Auth/DB/Storage) → AWS (RDS/S3/ECS)
Vercel 배포                → Vercel + AWS
Google Gemini              → OpenAI/Claude 전환
단일 앱 (src/)             → 모노레포 (Turborepo)
```

### 핵심 결정 사항

| 항목 | Phase 0-1 (MVP) | Phase 2+ (스케일업) |
|------|-----------------|-------------------|
| 프론트엔드 | Next.js 15 App Router | Next.js (유지) |
| 백엔드 | Supabase | NestJS + TypeORM |
| 데이터베이스 | Supabase PostgreSQL | AWS RDS PostgreSQL |
| 인증 | Supabase Auth | Supabase Auth (유지) 또는 NextAuth.js |
| AI 엔진 | Google Gemini (무료) | OpenAI GPT-4o / Claude API |
| 파일 저장 | Supabase Storage | AWS S3 + CloudFront |
| 캐시/큐 | - | Redis + BullMQ |
| 모바일 | - | React Native (Expo) |
| 클라우드 | Vercel | Vercel + AWS |
| 개발 구조 | 단일 앱 (src/) | 모노레포 (Turborepo) |

## 기술 선정 배경

### edu-system-architect 평가 (찬성: 8/10)

- **TypeScript 풀스택**: 한국 개발자 생태계에 매우 적합
- **PostgreSQL RLS**: 멀티테넌시 구현에 탁월
- **Mathpix + GPT/Claude**: 수학 OCR의 최선 조합
- **추가 권장**: 비동기 AI 채점, 성취기준 DB 구축

### edu-critical-analyst 평가 (비판: 2/10)

| 우려 사항 | 필수 완화책 |
|-----------|------------|
| 외부 API로 학생 데이터 해외 유출 | 익명화 레이어 구축 |
| 단일 장애점 위험 | 로컬 폴백 시스템 |
| 소형 학원 비용 전가 | 정액제 요금 모델 |
| AI 채점 신뢰성 | 교사 오버라이드 권한 필수 |

### 보안 규칙

- `NEXT_PUBLIC_` 접두사: 클라이언트에 노출되어도 안전한 값만 사용 (Supabase anon key, 앱 URL 등)
- `SUPABASE_SERVICE_ROLE_KEY`, `GEMINI_API_KEY` 등: 절대 `NEXT_PUBLIC_` 접두사 금지
- `.gitignore`에 `.env*` 필수 포함 (`.env.example` 제외)

### 환경변수 검증 예시

```typescript
// src/lib/env.ts
import { z } from 'zod'

const envSchema = z.object({
  NEXT_PUBLIC_SUPABASE_URL: z.string().url(),
  NEXT_PUBLIC_SUPABASE_ANON_KEY: z.string().min(1),

  GEMINI_API_KEY: z.string().min(1),
})

export const env = envSchema.parse({
  NEXT_PUBLIC_SUPABASE_URL: process.env.NEXT_PUBLIC_SUPABASE_URL,
  NEXT_PUBLIC_SUPABASE_ANON_KEY: process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY,
  SUPABASE_SERVICE_ROLE_KEY: process.env.SUPABASE_SERVICE_ROLE_KEY,
  GEMINI_API_KEY: process.env.GEMINI_API_KEY,
})
```

## Phase 전환 마이그레이션 가이드

### 전환 결정 기준

다음 조건 중 하나 이상을 충족하면 Phase 2+ 전환을 검토합니다:

| 지표 | 임계값 |
|------|--------|
| 월 매출 | 100만원 이상 |
| 동시접속자 | 100명 이상 |
| Supabase DB 용량 | 400MB 이상 (Free 한도 80%) |
| AI API 호출량 | Gemini 무료 한도의 70% 도달 |

### 마이그레이션 체크리스트

#### 1. 모노레포 전환 (2-3일)

- [ ] Turborepo + pnpm workspace 설정
- [ ] `src/` → `apps/web/src/` 이동
- [ ] `packages/` 구조 생성 (types, utils, validators, config)
- [ ] 공유 코드 추출 및 패키지화
- [ ] 빌드/린트 파이프라인 검증

#### 2. NestJS 백엔드 도입 (3-5일)

- [ ] `apps/api/` NestJS 프로젝트 초기화
- [ ] Next.js API Routes → NestJS 엔드포인트 마이그레이션
- [ ] TypeORM 엔티티 정의
- [ ] BullMQ 작업 큐 설정 (AI 채점 비동기 처리)

## 검증 방법

### 1. 기술 검증 (PoC)

| 검증 항목 | 검증 방법 | Phase |
|----------|----------|-------|
| Supabase Auth 소셜 로그인 | 카카오/네이버/Google 연동 테스트 | 0-1 |
| PostgreSQL RLS | 멀티테넌시 시나리오 테스트 | 0-1 |
| AI 문제 추출 | Gemini 프롬프트 최적화 | 0 |
| AI 채점 정확도 | 수학 200문제 정답률 측정 | 2+ |
| OCR 정확도 | Mathpix vs Google Vision 비교 | 3+ |

### 2. 성능 검증

- **부하 테스트**: k6, Artillery
- **목표**: 동시 접속 1,000명, 응답 시간 < 500ms
- **AI 채점 큐**: 초당 10건 처리 목표

### 3. 보안 검증

- OWASP Top 10 체크리스트
- SQL Injection, XSS 방어 확인
- 익명화 레이어 검증
- 개인정보보호법 준수 확인
